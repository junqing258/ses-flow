## 2. 文档目标

本文件用于为后续绘制系统原型图提供清晰、可执行的页面说明，重点解决以下问题：

- 系统需要绘制哪些页面
- 各页面分别服务于哪些角色
- 各页面承载什么业务目的
- 页面上应包含哪些关键区域
- MVP 阶段优先画哪些页面，二期预留哪些页面

本文件不替代系统 PRD，也不展开到字段级交互说明、弹窗级逻辑和高保真视觉设计。文档粒度控制在“页面清单 + 页面目的 + 关键区域”，重点用于帮助研发与实施团队在正式绘制原型前快速对齐系统页面范围。

## 3. 原型范围与输出原则

### 3.1 原型范围

本次原型描述覆盖两大终端：

- 后台管理端
- 现场作业端（PDA/工位终端）

其中，后台管理端用于承接流程编排、规则配置、波次管理、任务追踪、集成治理、监控告警和系统配置；现场作业端用于承接扫描入框、集包、打印/贴码和现场异常处理等作业场景。

### 3.2 输出原则

- 角色入口清晰：不同岗位登录后看到的入口和工作台边界明确
- 模块归属清晰：每个页面明确归属到具体模块
- 业务闭环清晰：原型页能够覆盖从接单到贴码以及异常处理闭环
- 绘制可执行：页面说明足以支撑线框图或低保真原型直接开画
- 先范围后细节：优先确认页面结构和导航关系，不提前陷入细节交互

## 4. 信息架构总览

系统原型建议采用“统一门户 + 角色工作台 + 模块页面”的组织方式。

### 4.1 后台管理端一级导航

后台管理端建议包含以下一级导航模块：

- 首页工作台
- 流程编排
- 规则中心
- 波次管理
- 作业任务中心
- 集成中心
- 监控告警
- 审计日志
- 系统配置

### 4.2 现场作业端入口

现场作业端建议按岗位拆分入口，避免一个复杂的通用首页。岗位入口建议包括：

- 扫描入框岗位入口
- 集包岗位入口
- 打印/贴码岗位入口
- 现场异常处理岗/班组长入口

### 4.3 角色工作台

系统建议至少具备以下角色化入口：

- 产品/实施工作台
- 运营主管工作台
- 运维/集成工作台
- 现场作业岗位直达入口

## 5. 角色入口页面说明

### 5.1 统一登录/租户选择页

页面目的：
作为系统统一入口，用于登录、租户切换、客户或仓上下文选择。

关键区域：

- 登录信息区
- 租户/客户选择区
- 仓选择区
- 角色提示区
- 最近访问入口区

### 5.2 后台首页工作台

页面目的：
作为后台用户统一首页，展示待办事项、核心业务状态、异常摘要和快捷入口。

关键区域：

- 角色欢迎区
- 待办事项区
- 波次与任务概览区
- 异常告警摘要区
- 常用功能快捷入口区

### 5.3 现场作业岗位入口页

页面目的：
为现场终端提供岗位选择，或作为不同岗位登录后的默认首页入口。

关键区域：

- 岗位入口区
- 当前仓/工位信息区
- 班次或当前状态区
- 快捷异常入口区

## 9. 原型绘制顺序建议

建议按三轮推进原型绘制：

### 9.1 第一轮

目标：
快速对齐系统主结构与核心闭环。

建议优先绘制：

- 统一登录/租户选择页
- 后台首页
- 流程模板列表页
- 流程模板设计页
- 规则集列表页
- 规则配置页
- 波次列表页
- 任务列表页
- 扫描入框首页/执行页
- 集包首页/执行页
- 打印贴码首页/执行页

### 9.2 第二轮

目标：
补齐异常处理、集成追踪和运维视角。

建议补充绘制：

- 波次详情页
- 任务详情页
- 系统连接列表页
- 接口调用记录页
